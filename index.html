A lightweight, tool-agnostic convention for making substantive AI use visible and auditable. Ask the AI tool to maintain a dated project-history file that records the request, inputs, actions, changes, outputs, and verification. Teams can use any suitable storage service, writing platform, version-control system, or AI assistant.

What This Guide Recommends

The goal is not to prescribe a particular product. It is to keep a durable record close to the project so collaborators can understand how AI contributed without reconstructing the work from a private chat.

  • Record substantive AI-assisted work, including inspection and decisions even when no project file changes.
  • Use dated, human-readable files that collaborators can inspect with ordinary tools.
  • Separate records by human collaborator and identify the AI tool that performed the work.
  • Include concrete inputs, files, commands, outputs, verification results, assumptions, and unresolved questions.
  • Store the history with the project when practical and follow the project's privacy, security, authorship, and version-control rules.

Preferred Example: Dropbox + Overleaf + Codex

Dropbox + Overleaf + Codex is the preferred example because collaborators can continue writing in Overleaf, Dropbox provides a synchronized local project folder, and Codex can edit the same files while maintaining the history record automatically.

  1. Synchronize the shared Overleaf project with Dropbox. Keep the existing folder structure; no reorganization is required.
  2. Install the shared Codex instructions. Copy the setup prompt and paste it into Codex. Codex should save the global instruction file automatically.
    Codex Setup
  3. Open the synchronized Overleaf project folder in Codex as the working directory.
  4. Prompt normally. After substantive work, Codex should create or append to the dated history file automatically.

Adapt It to Other Tools

The same convention works with other AI assistants, synchronized folders, document platforms, and repositories. Translate the setup details—not the transparency standard—to the tools your team already uses.

  • Another AI tool: place equivalent global or project instructions where that tool reads them. The included Claude Code setup is one example.
  • GitHub-centered Overleaf: work from a local clone connected to Overleaf and follow the repository's branch, commit, review, and synchronization rules.
  • Another platform: keep the history beside the project when practical, or in another shared, durable location agreed upon by the team.
  • Tools without file access: export or copy a concise activity record into the project history at the end of substantive work.

Project History Convention

When the project permits it, the AI tool should create or append to this file automatically:

project_history/Your_Name/YYYY-MM-DD by Agent.md

Use one folder per human collaborator and one file per date and AI tool. Record substantive inspection, decisions, and verification even when no other file changes.
